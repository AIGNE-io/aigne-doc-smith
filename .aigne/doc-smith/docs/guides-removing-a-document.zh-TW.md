# 移除文件

保持文件準確性不僅涉及新增內容，還需要進行整理。本指南詳細介紹如何使用 `aigne doc remove-document` 指令安全地從您的專案中刪除文件，確保您的文件集保持整潔和相關性。

## 概覽

`remove-document` 指令提供了一種互動式的方式，讓您從現有結構中選擇並刪除一個或多個文件。此指令的一個關鍵功能是它能夠處理連鎖刪除；移除一個父文件也會移除其所有的子文件。

此外，在選定的文件被移除後，該工具會自動掃描剩餘的檔案，尋找任何指向已刪除文件的損壞連結。然後，它會嘗試智慧地修復或移除這些連結，確保您整個文件集的完整性。

## 如何移除文件

要開始移除過程，請導覽至您專案的根目錄並執行以下指令：

```sh icon=lucide:terminal
aigne doc remove-document
```

您也可以使用較短的別名 `remove` 或 `rm`：

```sh icon=lucide:terminal
aigne doc rm
```

### 逐步流程

1.  **選擇文件：** 執行指令後，您會看到目前文件的列表，以階層式的樹狀結構顯示。您可以使用方向鍵在此列表中導覽，並透過按下空白鍵來選擇您希望移除的文件。

2.  **確認選擇：** 當您選取完所有要刪除的文件後，按下 `Enter` 鍵進行確認。如果您決定不移除任何文件，可以在未做任何選擇的情況下按下 `Enter` 鍵來取消操作。

3.  **連結驗證與修復：** 文件被刪除後，DocSmith 會自動掃描您剩餘的文件，尋找任何現在指向不存在檔案的連結。系統會提示您確認是否自動修復這些無效連結。

4.  **檢視摘要：** 最後，您的終端機中會顯示一份摘要，列出所有成功移除的文件，並詳細說明哪些文件的無效連結已被修復。

這個過程確保了移除檔案的直接性，並且不會在您的其他文件中留下損壞的引用。
