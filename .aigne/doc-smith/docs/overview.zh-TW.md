# 總覽

是否還在為了讓文件與不斷變化的程式碼庫保持同步而苦惱？AIGNE DocSmith 將整個流程自動化，分析您的原始程式碼以產生準確、結構化且多語言的文件，讓您能專注於打造優秀的軟體。

AIGNE DocSmith 是一款 AI 驅動的工具，可從您的專案原始程式碼中自動建立文件。它建基於 [AIGNE Framework](https://www.aigne.io/en/framework)，旨在產生能準確反映您程式碼庫的結構化、多語言文件。

該工具解決了手動建立文件時的常見挑戰，例如建立過程耗時、程式碼演進後文件很快就過時，以及不同章節之間缺乏一致性。透過將此過程自動化，DocSmith 有助於確保您的文件保持最新、準確且實用。

## 運作方式

DocSmith 透過分析您專案的原始程式碼來理解其結構、元件和功能。基於此分析，它會產生一套完整的文件，從高階指南到詳細的 API 參考資料都包含在內。

## 核心功能

DocSmith 提供了一系列功能，以處理從建立到發布的整個文件生命週期。

*   **AI 驅動產生**：分析您的程式碼庫，提出合乎邏輯的文件結構，並產生解釋您程式碼功能的內容。
*   **多語言支援**：將文件翻譯成 12 種語言，包括英文、中文（簡體）和日文。翻譯過程具備語境感知能力，以保持技術準確性。
*   **與 LLM 整合**：與各種大型語言模型（LLM）連接。預設情況下，它使用 [AIGNE Hub](https://www.aigne.io/en/hub)，這項服務讓您可以在 Google Gemini 和 OpenAI GPT 等模型之間切換，而無需個別的 API 金鑰。您也可以設定自己的 API 金鑰以直接存取供應商。
*   **智慧更新**：偵測您原始程式碼中的變更，並更新文件的相應章節。您也可以提供具體回饋以完善產生的內容。
*   **發布選項**：只需一個指令即可發布您建立的文件。您可以部署到官方的 DocSmith 平台，或執行您自己的 [Discuss Kit](https://www.web3kit.rocks/discuss-kit) 執行個體。Discuss Kit 是一項用於託管和顯示文件的服務。

## 可用指令

DocSmith 透過命令列介面操作。下表總結了主要指令及其功能。

| 指令 | 說明 |
| :--- | :--- |
| `create` | 從您的來源檔案建立一套新文件。 |
| `update` | 根據程式碼變更或新的回饋修改現有文件。 |
| `localize` | 將文件翻譯成 12 種支援的語言中的一種或多種。 |
| `publish` | 將您的文件部署到一個可公開存取的 URL。 |
| `evaluate` | 評估您所建立文件的品質與完整性。 |
| `history` | 檢視對您的文件所做的更新歷史記錄。 |
| `chat` | 啟動互動模式會話以產生和管理文件。 |
| `clear` | 移除已建立的檔案、設定和快取資料。 |
| `init` | 引導您完成一個互動過程，以建立初始設定檔。 |
| `prefs` | 管理用於文件建立的已儲存偏好設定和組態。 |
